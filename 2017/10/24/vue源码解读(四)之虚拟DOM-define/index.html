Posted on 2017-10-24

vue初始化完成后，就是的渲染和更新dom了，但是讲到这部分前，必须要先认识虚拟DOM

Vue.js将DOM抽象成一个以JavaScript对象为节点的虚拟DOM树，以VNode节点模拟真实DOM，可以对这颗抽象树进行创建节点、删除节点以及修改节点等操作，在这过程中都不需要操作真实DOM，只需要操作JavaScript对象，大大提升了性能。修改以后经过diff算法得出一些需要修改的最小单位，再将这些小单位的视图进行更新。这样做减少了很多不必要的DOM操作，大大提高了性能。

virtual dom 另一个重大意义就是提供一个中间层，js去写ui，ios安卓之类的负责渲染，就像reactNative一样。

在 HTML DOM 中，所有事物都是节点。DOM 是被视为节点树的 HTML。所以看虚拟dom先看虚拟节点的定义，它是对真实Dom的一层抽象，而不依赖某个平台

VNode分类:

- EmptyVNode 没有内容的注释节点
- TextVNode  文本节点
- ElementVNode 元素节点
- ComponentVNode 组件节点
- CloneVNode 克隆节点
- ...

大致了解了vnode的定义和构成，下一步就是创建元素
