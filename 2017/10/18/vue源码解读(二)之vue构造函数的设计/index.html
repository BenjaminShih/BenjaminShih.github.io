vue源码解读(二)之vue构造函数的设计

Posted on 2017-10-18
Words count in article:

上一篇中我们找到了vue构造函数的定义：

简单的定义了vue构造函数，引入依赖，将vue构造函数作为参数经过initMixin(),stateMixin(),eventsMixin(), lifecycleMixin(), renderMixin()五个函数处理后，导出此函数。

通过查看引入依赖的五个文件，我们可以综合起来看，构造函数经过了这些处理

经过这五个步骤的混入，Vue构造函数的原型对象挂载了很多方法，比如我们业务中常用的emit、destory方法等等，但是这并没完，按照上一篇中的步骤往回走，下一个处理Vue构造函数的是src/core/index.js文件

这里除了在Vue挂载version和Vue.prototype上挂载$isServer和$ssrContext属性，还有initGlobalAPI()方法处理

经过这一步，我们又见到了熟悉的options，use(),directive等方法和属性，通过上一篇文章我们知道，下一步就是要经过src/platforms/web/runtime/index

总结来讲src/platforms/web/runtime/index总共做了三件事：

1、覆盖 Vue.config 的属性，将其设置为平台特有的一些方法
2、Vue.options.directives 和 Vue.options.components 安装平台特有的指令和组件
3、在 Vue.prototype 上定义 patch 和 $mount

最后一个处理 Vue 的文件就是入口文件 src/platform/web/runtime-with-compiler.js 了

其实就做了两件事，重写src/platforms/web/runtime/index中的$mount函数和在Vue上挂载compile

至此，我们还原了Vue的构造函数，前一篇文章从入口逆流而上找到定义源头，这篇我们再顺流而下捋清Vue构造函数的设计，总结如下

1、Vue.prototype 下的属性和方法的挂载主要是在 src/core/instance 目录中的代码处理的
2、Vue 下的静态属性和方法的挂载主要是在 src/core/global-api 目录下的代码处理的
3、src/platform/web/runtime.js 主要是添加web平台特有的配置、组件和指令
4、src/platform/web/runtime-with-compiler.js给Vue的 $mount 方法添加 compiler 编译器，支持 template。
