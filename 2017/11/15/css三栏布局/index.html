css三栏布局

Posted on 2017-11-15
Words count in article:

1. 表格布局

优点： 代码简单易懂
缺点： 无法设置栏间距

2. float + margin流体方案

左右模块各自向左右浮动，并设置中间模块的 margin 值利用块级元素的特性使中间模块宽度自适应。

缺点: 内容较多时候先加载侧边栏

3. float + BFC 方案

BFC 区域，不会与浮动元素重叠，这里与1中的唯一区别是主体利用BFC而不是margin进行流体定位

缺点也是侧边栏先加载

4. 双飞翼布局方案

利用的是浮动元素 margin 负值的应用

优点: 主体内容先加载

缺点: html结构比较复杂

5. 圣杯布局方案

优点：也是主体内容先加载，html结构简单
缺点：css代码较为复杂，在宽度过小的时候布局会乱

6. flex 布局

优点：代码简洁
缺点：兼容性问题

  1. position + margin 定位方案

优点: 代码简单

缺点: 

- 如果中间栏含有最小宽度限制，或是含有宽度的内部元素，当浏览器宽度小到一定程度，会发生层重叠的情况。
- 侧边栏无法撑起高度
